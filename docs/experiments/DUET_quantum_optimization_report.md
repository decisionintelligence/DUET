# DUET 量子模块优化实验报告

**实验日期**: 2026-03-20
**数据集**: ETTh1 (Electric Transformer Temperature Hourly)
**评测指标**: MSE_norm, MAE_norm, RMSE_norm

---

## 1. 实验背景与目标

### 1.1 研究动机
- 探索量子计算启发的时间序列预测方法
- 验证量子 OTOC (Out-of-Time-Order Correlator) 机制的有效性
- 通过渐进式优化提升模型性能

### 1.2 核心创新
在 DUET 模型中引入 Quantum OTOC Block：
- **复值量子态表示**: 将输入特征映射为复数表示
- **Cayley 酉变换**: 通过酉矩阵实现幺正演化
- **OTOC 权重计算**: 利用量子退相干度量作为注意力权重
- **Highway Gate 融合**: 与标准 Transformer 特征自适应融合

## 4. 各版本详细分析

### 4.1 Baseline (原版 DUET)
**结果**: MSE=0.3520, MAE=0.3843, RMSE=0.5810 (最佳)

**特点**:
- 纯 Transformer 架构
- 无量子模块
- 作为所有实验的基准

### 4.2 v2 (量子 + Highway Gate)
**结果**: MSE=0.3629 (+3.1%)

**改进**:
- 引入量子 OTOC Block
- 使用 Highway Gate 融合量子与 Transformer 特征

**分析**: 轻微性能下降，但证明了量子模块的可整合性

### 4.3 v4 (量子 + Highway Gate + 调优)
**结果**: MSE=0.3672 (+4.3%)

**改进**:
- dropout=0.5 (更高正则化)
- k=2 (专家数量增加)

### 4.4 v7 (保守优化版)
**结果**: MSE=0.3726 (+5.8%)

**保守设计**:
- alpha=0.8 (量子模块影响最小化)
- 保留原始 OTOC 计算
- 简化 SE 门控
- 去除复杂增强

**分析**:
- 保守设计并未带来性能提升
- 表明问题可能不在于量子模块的"强度"
- 暗示可能需要全新的设计思路

### 4.5 v6 (激进优化版)
**结果**: MSE=0.3804 (+8.1%)

**激进设计**:
- alpha=0.3 (量子模块主导)
- 李雅普诺夫近似
- 增强预测头 (瓶颈结构)
- 特征调制

**分析**: 最差表现，说明过度设计反而有害

## 5. 关键发现与结论

### 5.1 主要结论

1. **原版 Baseline 仍然最优**: 在 ETTh1 数据集上，所有量子模块版本都未能超越原版

2. **性能差距**: 量子模块版本与 Baseline 的 MSE 差距在 3-30% 之间

3. **复杂度负相关**: 模型复杂度增加与性能提升呈负相关

### 5.2 可能原因分析

#### 5.2.1 数据集特性
- **ETTh1 难度高**: 变压器温度预测具有复杂的非线性动力学
- **数据量有限**: 相对于模型复杂度，数据可能不足
- **分布复杂**: ETTh1 的时间模式可能不适合量子态表示

#### 5.2.2 量子模块问题
- **物理模型不匹配**: 简化的酉变换可能无法捕捉真实物理动力学
- **梯度流动问题**: 酉矩阵约束可能限制梯度传播
- **特征空间不匹配**: 复值表示可能不适合实值时序数据

#### 5.2.3 融合策略问题
- **Highway Gate 局限性**: 简单的门控可能不足以平衡两种特征
- **自适应融合过拟合**: 更多参数可能在小数据集上过拟合

### 5.3 改进建议

#### 方案 A: 数据集替换
在更简单的数据集（如 ETTm2）上验证量子模块有效性

#### 方案 B: 量子模块重构
- 考虑使用更轻量的量子启发机制
- 探索量子卷积而非酉变换
- 尝试量子注意力机制

#### 方案 C: 训练策略优化
- 预训练 + 微调策略
- 多任务学习
- 课程学习

#### 方案 D: 特征工程
- 量子模块作为辅助特征
- 并行而非串行架构
- 特征解耦与重组
